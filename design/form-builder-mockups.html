Option 1

Stacked Row Cards

The form is a vertical list of rows. Each row card holds 1–4 columns, and each column holds one field. "Add row" appends a new card; "+ Column" splits the row. Closest to the current UI, easiest to build. No X/Y coordinates at all — position comes from row/column order.

Option 2

Field Palette + Drop Canvas

A palette of field types on the left (Text, Checkbox, Dropdown, Date, Signature). Drag a field into a row on the canvas — dropping beside an existing field creates a new column in that row; dropping below creates a new row. Feels like classic form-builder tools (Typeform/Google Forms style).

Option 3

WYSIWYG Page Canvas

The builder shows the actual white PDF page (US Letter proportions) exactly as it will export. A toolbar adds rows, columns, and field types. Click a field on the page to select it — a floating mini-toolbar switches its type or splits its row. What you see is literally the exported PDF.

Option 4

Grid / Spreadsheet Table

Rows and columns as a literal table: each table row = a PDF row, each cell = a field. Toolbar buttons add rows or columns; empty cells show a dashed "+" placeholder. Very dense and fast for keyboard-heavy editing of large forms — no dragging needed.

Option 5

Compact Canvas + Right Inspector

Left: a compact structural view — numbered rows containing field pills; click "+" inside a row to add a column, or "+ Add row" below. Right: an inspector panel that edits whatever is selected (label, type, options, height, column span). Keeps the field list uncluttered because all settings live in the inspector.

All five options share the same underlying model: the form is rows → columns → fields (text, checkbox, dropdown, and optionally date/signature). Export walks rows top-down, computing Y from row order and X/width from column count — manual X/Y coordinates go away entirely. Options differ only in how you edit that structure: 1 is the simplest to implement, 3 gives the truest preview, 5 scales best for long forms.
